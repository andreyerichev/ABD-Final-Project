# Разработка модели предсказания релевантных продуктов и платёжных методов для действующих клиентов интернет-эквайринга с использованием методов машинного обучения

**ФИО студента:** Еричев Андрей Валерьевич

**ФИО научного руководителя:** Кубаева Ассоль Рауфжоновна

### План
1. Описание бизнес-задачи (янв 2026)
На данном этапе будет сформулирована бизнес-задача повышения эффективности кросс-сейла продуктов и платёжных методов среди действующих клиентов интернет-эквайринга. Будет описан текущий процесс подключения дополнительных продуктов и существующие ограничения, связанные с ручным или неформализованным выбором клиентов. Также будет определено, каким образом использование методов машинного обучения может помочь автоматизировать процесс выявления клиентов, для которых определённые продукты или платёжные методы являются наиболее релевантными.

2. Обзор возможных подходов к решению задачи (янв 2026)
На этом этапе будет проведён обзор возможных подходов к решению задачи кросс-сейла в финтех-продуктах. По итогам анализа будет выбран наиболее подходящий подход с учётом структуры данных, особенностей данных и бизнес-процессов, а также требований к интерпретируемости модели.

3. Обоснование выбранного решения (янв 2026)
Будет проведён анализ потенциальной бизнес-ценности применения модели машинного обучения для задачи кросс-сейла и ключевые бизнес-метрики, по которым будет оцениваться эффективность предложенного решения.

4. Сбор данных и подготовка датасетов (фев 2026)
На данном этапе будут собраны и объединены данные, необходимые для построения моделей машинного обучения. Также будет выполнена предварительная обработка данных, включая очистку, обработку пропусков и формирование признаков.

5. Разведочный анализ данных (EDA) (фев 2026)
Будет проведён разведочный анализ данных с целью изучения структуры датасета, распределения признаков и особенностей целевых переменных. Результаты анализа будут использованы для дальнейшего построения моделей.

6. Построение baseline-моделей, выбор метрик качества (мар 2026)
На данном этапе будут построены базовые модели бинарной классификации для предсказания вероятности подключения отдельных продуктов и платёжных методов. Также будут выбраны метрики качества моделей, отражающие специфику бизнес-задачи и дисбаланс классов. Результаты baseline-моделей будут использоваться как отправная точка для оценки эффективности более сложных моделей.

7. Обучение продвинутых моделей (мар 2026)
Будут обучены более сложные модели машинного обучения с использованием алгоритмов, подходящих для задач табличных данных. Планируется провести настройку гиперпараметров и сравнение различных моделей по выбранным метрикам качества. По результатам экспериментов будет выбрана итоговая модель или набор моделей, демонстрирующих наилучшие результаты.

8. Оценка экономического эффекта от подключения различных комбинаций продуктов и платёжных методов (апр 2026)
На данном этапе будет проведена оценка потенциального экономического эффекта от внедрения разработанной модели. Планируется рассчитать ожидаемую дополнительную выручку, которая может быть получена за счёт более точного определения клиентов, для которых релевантно подключение дополнительных продуктов и платёжных методов. Также будет рассмотрено влияние различных комбинаций продуктов на потенциальный финансовый результат.

9. Разработка MVP-решения и описание сценариев использования итоговой модели (апр 2026)
На заключительном этапе будет разработан прототип (MVP) решения, демонстрирующий применение построенной модели на практике. Также будут описаны возможные сценарии использования модели в бизнес-процессах компании, включая формирование списков клиентов для кросс-сейла. Отдельное внимание будет уделено возможным вариантам интеграции модели в существующую инфраструктуру.
